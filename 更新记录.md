# 项目更新记录

## 版本对比：qybs2.8 → qybs2.9wcc

### 1. 目录结构变更

#### 新增目录：
- `logs/` - 日志文件目录
- `services/` - 服务层目录
- `static/` - 静态资源目录（包含 js/ 和 styles.css）
- `templates/` - 模板文件目录（替代旧版本的 web/ 目录）

#### 目录调整：
- 模板文件从 `web/` 目录迁移到 `templates/` 目录

### 2. 新增文件

#### 核心文件：
- `logger.py` - 日志配置和工具
- `routes/like_routes.py` - 点赞功能路由
- `services/file_service.py` - 文件服务
- `services/news_service.py` - 新闻服务
- `static/js/comments.js` - 评论相关前端脚本
- `static/styles.css` - 全局样式文件
- `templates/news_detail.html` - 新闻详情页模板

### 3. 功能变更

#### 新增功能：
1. **点赞功能**
   - 支持用户对新闻进行点赞/取消点赞
   - 显示新闻的点赞数量
   - 记录用户的点赞状态

2. **用户系统**
   - 新增用户表结构
   - 支持用户注册和登录

3. **日志系统**
   - 请求和响应日志记录
   - 应用初始化日志
   - 日志文件存储

4. **安全性改进**
   - 新增登录尝试限制表
   - 防止暴力破解攻击

5. **服务层架构**
   - 分离业务逻辑到服务层
   - 提高代码可维护性

### 4. 数据库结构变更

#### 新增表：
1. **users 表**
   - 存储用户信息
   - 字段：id, username, password, created_at

2. **likes 表**
   - 存储用户点赞记录
   - 字段：id, user_id, news_id, created_at
   - 唯一约束：user_id + news_id

3. **login_attempts 表**
   - 存储登录尝试记录
   - 字段：id, ip_address, username, attempts, last_attempt, locked_until
   - 唯一约束：ip_address + username

#### 现有表调整：
- **admin 表**：改进密码加密方式，对用户名也进行加密

### 5. 代码改进

#### 主应用文件 (app.py)：
- 新增日志记录器
- 添加请求/响应日志中间件
- 注册新增的点赞路由
- 改进应用初始化日志

#### 路由文件：
- 新增 `like_routes.py` 实现点赞相关接口
- 其他路由文件保持不变

#### 数据库文件 (db.py)：
- 新增用户、点赞、登录尝试表的创建
- 改进管理员账号创建逻辑

### 6. 前端改进

#### 模板文件：
- 新增 `news_detail.html` 新闻详情页
- 调整其他模板文件以适应新的目录结构

#### 静态资源：
- 新增 `comments.js` 处理评论相关交互
- 新增 `styles.css` 全局样式

### 7. 其他变更

- 保留了旧版本的所有测试文件
- 保持了与旧版本相同的 API 接口，确保向后兼容
- 改进了错误处理和异常捕获

### 8. 总结

qybs2.9wcc 版本在保留原有功能的基础上，新增了用户系统、点赞功能、日志系统等重要特性，同时优化了代码结构和安全性。这些变更使得系统更加完善，用户体验更加丰富，同时为后续的功能扩展奠定了基础。
