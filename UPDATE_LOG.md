# 七一报社项目更新记录（qybs2.7 → qybs2.8wcc）

## 1. 架构与数据存储

### 1.1 数据存储升级
- **qybs2.7**: 使用JSON文件存储数据（json/文件夹）
- **qybs2.8wcc**: 迁移到SQLite数据库（site.db）
- **迁移工具**: 新增 `migrate_json_to_sqlite.py` 脚本，支持从旧JSON数据迁移到新数据库

### 1.2 目录结构调整
- **qybs2.7**: 
  - `html/`: 前端页面
  - `json/`: 数据文件
  - `static/`: 静态资源
- **qybs2.8wcc**:
  - `web/`: 前端页面（替代原html/）
  - `uploads/`: 媒体文件存储
  - `routes/`: 路由模块（重构）

## 2. 后端功能升级

### 2.1 路由模块重构
- **qybs2.7**: 简单路由结构
- **qybs2.8wcc**: 更规范的路由命名
  - `ad_routes.py`: 广告管理
  - `admin_routes.py`: 管理员功能
  - `auth_routes.py`: 认证功能
  - `comment_routes.py`: 评论功能（新增）
  - `discuss_routes.py`: 讨论功能
  - `download_routes.py`: 下载功能（新增）
  - `news_routes.py`: 新闻管理
  - `pages_routes.py`: 页面路由（新增）

### 2.2 数据库管理
- **新增 `db.py`**: 提供SQLite数据库连接和初始化功能
- **表结构**:
  - `news`: 新闻数据
  - `comments`: 评论数据
  - `ads`: 广告数据
  - `discuss`: 讨论数据
  - `admin`: 管理员账号

### 2.3 安全性提升
- **新增 `auth_utils.py`**: 认证工具
- **密码加密**: 使用Argon2id算法加密存储管理员密码
- **默认管理员**: 自动创建默认管理员账号

## 3. 前端功能增强

### 3.1 页面结构优化
- **qybs2.7**: 基础管理页面
- **qybs2.8wcc**: 增强版管理界面
  - 媒体库功能：支持上传和管理媒体文件
  - 内容管理：支持下载和清理失效新闻
  - 广告管理：独立的广告管理界面

### 3.2 用户体验改进
- **消息系统**: 统一的消息显示机制
- **媒体预览**: 上传文件时的实时预览
- **响应式设计**: 优化的界面布局
- **Bug修复**: 修复了admin页面无法返回首页的bug

### 3.3 代码质量提升
- **CSS优化**: 移除内联样式，使用统一的CSS类
- **JavaScript功能增强**: 改进的交互逻辑

## 4. 配置与部署

### 4.1 配置文件更新
- **qybs2.7**: 基础配置
- **qybs2.8wcc**: 更完善的配置
  - 使用绝对路径
  - 增加ADMIN_USERNAME和ADMIN_PASSWORD配置
  - 统一的文件路径管理

### 4.2 部署工具
- **新增 `fix_admin_password.py`**: 修复管理员密码工具
- **测试文件**: 新增多个测试脚本，确保功能正常

## 5. 新增功能

1. **媒体库管理**: 支持上传、预览和管理媒体文件
2. **内容下载**: 支持批量下载新闻内容
3. **失效新闻清理**: 自动清理文件不存在的新闻记录
4. **广告管理**: 独立的广告添加和管理功能
5. **评论系统**: 新闻评论功能
6. **讨论区**: 用户讨论功能

## 6. 技术栈更新

- **后端**: Flask + SQLite（替代JSON文件存储）
- **前端**: HTML5 + CSS3 + JavaScript
- **安全**: Argon2id密码加密
- **数据迁移**: 支持从旧版本JSON数据迁移

## 7. 兼容性与迁移

- **数据迁移**: 提供完整的JSON到SQLite迁移工具
- **向后兼容**: 保持API接口的一致性
- **部署便捷**: 简化部署流程，支持快速启动
